本文介绍匿名类。

# 解决什么问题

对于一次性的类，如果按照传统的方式定义一个单独的类文件，然后 import 使用，比较麻烦。

匿名类可以直接在当前类创建一个类出来，更加高效。

# 如何使用

详见 AnnoyClass.

# 注意事项

1. 因为匿名类是匿名的，所以无法在匿名类中定义构造函数，不过可以通过初始化块 `{}` 来初始化示例。
2. 匿名类实例创建出来都是通过接口、父类访问的，因此匿名类中新加的字段或者方法，是无法在外部访问的。
3. 匿名类可以访问外部类的实例成员和静态成员，以及临时的 final 变量。与 go 中的闭包类似。

# 与 lambda 的区别

lambda 只能用来实现一个函数的接口(函数式接口)，而匿名类可以继承其他类，也可以实现拥有多个函数的接口。
